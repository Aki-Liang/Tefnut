# 设计：漫画库路径目录选择弹窗

日期：2026-07-03
状态：已确认

## 背景与目标

设置页「漫画库路径」目前靠手动输入服务器绝对路径添加库目录，容易输错、
对不熟悉服务器文件系统布局的用户不友好。目标：用一个**服务器端目录浏览
弹窗**取代手动输入 —— 用户在弹窗中逐级浏览 `allowedRoots` 内的目录并选定，
弹窗内提供路径输入框用于快速跳转。

浏览器原生文件夹选择框（`webkitdirectory`）选的是访问者本机文件夹且拿不到
绝对路径，对服务器端目录无效，故必须走服务器端列目录 API + 自绘弹窗方案
（同 Jellyfin / Komga 的做法），并遵循本项目「选择器一律自绘 格/PANEL 组件、
不用原生控件」的约定。

## 交互决策（已与用户确认）

- 表单中的路径文本输入框**移除**，只保留弹窗选择这一种方式。
- 弹窗内提供**快速跳转输入框**：粘贴/输入绝对路径回车直达。
- 显示名输入框与「添加」按钮保留在表单中，显示名仍自动取所选目录末段。
- 隐藏目录（`.` 开头）不显示。

## 后端

新文件 `internal/server/api_fs.go`（+ `api_fs_test.go`）。

### 路由

`GET /api/fs/dirs`，注册在 `server.go` 的 `/api` 组中，与其余设置 API 并列。

### 行为

- **无 `path` 参数或为空**：返回允许的根列表。对每个 `allowedRoots` 条目做
  存在性检查，跳过不存在/不可读的根。响应：
  `{ "roots": [{ "name": <末段>, "path": <绝对路径> }] }`
- **带 `path=/abs/dir`**：
  1. 要求是绝对路径，否则 400。
  2. 用现有 `PathWithinRoots(path, s.allowedRoots)` 校验；返回错误或 `false`
     一律 400（fail closed），错误消息与 `apiAddPath` 一致：「目录必须在允许
     的库根之内」。
  3. `os.ReadDir` 列出**直接子目录**：只要目录（含指向目录的符号链接）、
     跳过 `.` 开头的隐藏目录、跳过读取出错的条目，按名称排序。
  4. 响应：`{ "path": <规范绝对路径>, "parent": <上级路径或 "">, "dirs": [{ "name", "path" }] }`。
     `path` 本身就是某个根（`PathWithinRoots(parent) == false`）时 `parent` 为空串，
     前端此时「上级」退回根列表视图。
- 目录本身不可读 → 400，带用户可读消息。

### 安全边界

与 `apiAddPath` 完全一致：只读、只暴露目录名、只在 `allowedRoots` 之内；
符号链接经 `EvalSymlinks` 解析后再判定归属（由 `PathWithinRoots` 保证），
`..` 穿越与根外路径均被拒绝。不暴露文件、大小、权限等任何额外信息。

## 前端

### 表单改动（`settings.html`）

`#np-path` 文本输入框移除，替换为触发按钮：

```html
<button type="button" id="np-pick" class="pick-trigger" data-path="">选择库目录…</button>
<input id="np-name" placeholder="显示名（默认取末段）" autocomplete="off">
<button>添加</button>
```

- 未选择时按钮显示占位文案「选择库目录…」；选定后显示所选绝对路径，
  选中值存于 `data-path`。
- 提交时从 `data-path` 取路径，为空则不提交；POST `/api/settings/paths`
  逻辑与服务器校验完全不变。
- 显示名自动填充逻辑保留：选定目录后若用户未手动改过名字，自动填入末段。

### 弹窗（新文件 `web/static/js/dirpicker.js` + `app.css` 增补）

自绘模态弹窗，视觉与 `dd-menu` 同一套语言：panel 底色、2px `--line` 描边、
硬偏移阴影、半调网点背景，配全屏半透明遮罩。结构：

- **顶部**：标题「选择库目录」+ 当前路径显示 + 「上级」按钮。
- **快速跳转输入框**：输入绝对路径回车 → 请求该路径；失败（不合法/越界/
  不存在）时在弹窗内联显示错误提示，不用 `alert`。
- **中部**：子目录列表，单击进入下一级；为空时显示「无子目录」。
- **底部**：「选定当前目录」+「取消」。根列表视图下「选定当前目录」不可用
  （尚未进入任何目录时无可选路径；单根时初始即进入该根，可直接选定）。
- **状态流转**：单个根 → 打开即进入该根；多个根 → 先显示根列表。
  「上级」在根目录处退回根列表（多根）或禁用（单根）。
- **关闭方式**：取消按钮、Esc、点击遮罩。确认后回填表单触发按钮并关闭。
- 请求失败（网络/500）在弹窗内联提示，可重试。

## 错误处理汇总

| 场景 | 行为 |
| --- | --- |
| 快速跳转到非绝对路径/根外/不存在的路径 | 400 → 弹窗内联错误提示，停留原视图 |
| 列目录 API 网络或服务器错误 | 弹窗内联提示「加载失败」，当前视图不变 |
| 所有根都不存在 | 根列表为空 → 弹窗提示检查 `library.allowedRoots` 配置 |
| 未选路径点「添加」 | 不发请求（与现状对空路径的处理一致） |

## 测试

`internal/server/api_fs_test.go`，沿用 `server_test.go` 的 httptest 模式：

1. 无参数 → 返回全部存在的根；不存在的根被跳过。
2. `path=某根` → 返回其子目录，隐藏目录被跳过，只含目录不含文件。
3. `path=子目录` → `parent` 指向上级；`path=根` → `parent` 为空。
4. `path=../` 穿越、根外绝对路径 → 400。
5. 相对路径、不存在的路径 → 400。
6. 指向根外的符号链接目录 → 不可进入（400）。

前端 `dirpicker.js` 保持纯 DOM + fetch 的薄逻辑，无构建链，不单独加测试
（与现有 `settings.js` 等一致）；行为通过手动验证。

## 不做的事（YAGNI）

- 不支持在弹窗中新建目录（库应是已存在的收藏目录）。
- 不显示文件、大小、修改时间。
- 不做目录列表的键盘方向键导航（仅 Esc 关闭 + 回车跳转），需要时再加。
