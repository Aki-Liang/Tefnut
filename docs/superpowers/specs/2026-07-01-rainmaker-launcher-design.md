# `rainmaker` вЂ” Self-Contained One-Command Launcher Design

**Goal:** Replace `start.sh` (which depends on the repo's committed `docker-compose.yml` + `.env`) with `rainmaker`, a self-contained launcher that generates its own `docker-compose.yml` in the current directory and starts Tefnut from the published GHCR image вЂ” runnable as a single `curl вЂ¦ | bash` command with no repo checkout.

**Architecture:** One executable script `rainmaker` at the repo root (rewritten from `start.sh`, renamed). It has no dependency on any committed compose or env file: it prompts (reading `/dev/tty` so it works under `curl | bash`), **writes a `docker-compose.yml` with values baked in**, and launches with `docker compose up -d` (which auto-pulls the public image). The committed `docker-compose.yml` and `.env.example` are removed вЂ” the launcher is the single source of the compose definition.

**Tech Stack:** Bash, Docker Compose v2, the public multi-arch GHCR image `ghcr.io/aki-liang/tefnut:latest`.

## Global Constraints

- **Self-contained:** `rainmaker` does not read a committed compose or `.env`; it generates `./docker-compose.yml` itself, with the library path / port / timezone baked in.
- **One-command usable:** the README's primary path is `curl -fsSL https://raw.githubusercontent.com/Aki-Liang/Tefnut/main/rainmaker | bash`. Interactive prompts therefore read from `/dev/tty` (stdin is the curl pipe). A non-interactive form passes the library path as an argument: `вЂ¦ | bash -s -- /path/to/comics`.
- **Robust launch:** no separate hard-failing `docker compose pull` step (that was the reported failure mode). `docker compose up -d` auto-pulls the public image; on failure the script prints an actionable message (notably: a stale `docker login ghcr.io` в†’ `docker logout ghcr.io`).
- **Runs in the user's CWD:** the generated `docker-compose.yml` lands in the current directory (recommend running in an empty dir); the script does **not** `cd` to its own location.
- **Public image, pull (not build):** `image: ghcr.io/aki-liang/tefnut:latest`; no `--build` in the happy path.
- **Fixed mounts:** `<library>:/comics:ro` and named volume `tefnut-data:/data`. Port default 8086; TZ detected from the host, fallback `Asia/Shanghai`.
- **No change to** the Go code, Dockerfile, CI workflow, `deploy/config.yaml`, or the 1h scan default.

## Components

### 1. `rainmaker` (repo root, executable; replaces `start.sh`)

Bash, `set -euo pipefail`. Flow:
1. **Preflight:** `docker` + `docker compose` (v2) present, else error + install pointer, exit non-zero.
2. **`/dev/tty` prompt helper** `ask(prompt, default)`: prints the prompt and reads the answer from `/dev/tty` (so `curl | bash` prompts work); empty answer в†’ default; if there is no readable `/dev/tty`, returns the default without prompting.
3. **Library path:** from `$1` if given; else prompt (loop until an existing directory; expand a leading `~`). If neither a `$1` nor a tty is available (fully non-interactive), error telling the user to pass the path as an argument.
4. **Port** (default 8086, validated numeric 1-65535) and **timezone** (default detected from `/etc/localtime`, fallback `Asia/Shanghai`), via `ask`.
5. **Generate `./docker-compose.yml`** with the three values baked in:
   ```yaml
   services:
     tefnut:
       image: ghcr.io/aki-liang/tefnut:latest
       container_name: tefnut
       ports:
         - "<PORT>:8086"
       volumes:
         - <LIBRARY_PATH>:/comics:ro
         - tefnut-data:/data
       environment:
         - TZ=<TZ>
       restart: unless-stopped
   volumes:
     tefnut-data:
   ```
6. **Launch:** `docker compose up -d`. On non-zero exit, print the actionable failure message (ghcr reachability / stale login в†’ `docker logout ghcr.io`) and exit non-zero.
7. **Report:** poll `http://127.0.0.1:<port>/healthz` (if `curl` present), print the generated file's path, `docker compose ps`, and `http://localhost:<port>` (note first-run scan can take a few minutes).

## Data Flow

`rainmaker` в†’ prompts (`/dev/tty`) or `$1` в†’ generates `./docker-compose.yml` (baked values) в†’ `docker compose up -d` pulls the public image and starts the container в†’ the user browses `http://localhost:<port>`. The user is left with a plain, editable `docker-compose.yml` they can manage with `docker compose {down,up,logs}` afterward.

## Error Handling

- **docker / compose missing** в†’ message + install link, non-zero exit.
- **Library path invalid** в†’ re-prompt (interactive) or error+exit (arg form).
- **No tty and no path argument** в†’ error instructing the `bash -s -- /path` form.
- **`docker compose up -d` fails** (image pull / auth) в†’ actionable message: check ghcr.io reachability; if a prior `docker login ghcr.io` has expired, `docker logout ghcr.io` and retry.
- **Port already in use** в†’ Docker's own bind error surfaces from `up -d`; the script's message points the user to change the port.

## Testing

- **Static:** `bash -n rainmaker`; `shellcheck rainmaker` if available.
- **Generated compose:** run `rainmaker` non-interactively (`rainmaker <sampledir>`) in a temp dir with a stubbed docker (or just up to generation) and assert the emitted `docker-compose.yml` renders under `docker compose config` with the right image/port/volume/TZ.
- **Full one-command smoke (controller, Docker daemon):** in a temp dir, run `bash rainmaker <sample-comics>` вЂ” it generates `docker-compose.yml`, `docker compose up -d` pulls the **real public GHCR image**, `/healthz` returns 200, the browse page lists the sample, a page serves; then tear down (`docker compose down -v`, remove the temp dir). This exercises the real pull path end-to-end (the image is now published).
- No Go code changes; the Go suite is unaffected.

## Out of Scope

- Windows / PowerShell launcher.
- Uninstall/reset script.
- Publishing `rainmaker` to Homebrew or a package manager.
- Reverse proxy / TLS / auth.
- Keeping a committed `docker-compose.yml` as a fallback (removed by decision вЂ” single source is the launcher).
